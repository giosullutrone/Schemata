# Schemata Audit — 2026-03-05 (v2)

Second audit pass after completing all items from the first audit and implementing QoL improvements (image support, drag-drop, copy/paste, edge fixes, group node fixes).

## Bugs

### 1. `localStorage` access unprotected
- **Files:** `src/App.tsx`, `src/store/useCanvasStore.ts`
- **Issue:** `localStorage.getItem`/`setItem` can throw in private browsing mode or when storage quota is exceeded.
- **Fix:** Wrap all `localStorage` calls in try/catch with sensible fallbacks.
- **Severity:** Medium

### 2. `saveAllFiles()` race condition
- **File:** `src/store/useCanvasStore.ts`
- **Issue:** Reads the files snapshot at the start, then awaits sequential writes. If the user edits during save, the write uses stale data for files saved later in the loop.
- **Fix:** Snapshot each file's data immediately before its individual write, or lock edits during save.
- **Severity:** Medium

### 3. Viewport floating-point drift
- **File:** `src/store/useCanvasStore.ts` (`saveViewport`)
- **Issue:** Uses strict `===` to compare viewport values. Tiny floating-point differences (e.g., `1.0000000001` vs `1.0`) cause unnecessary dirty-file marks and saves.
- **Fix:** Use epsilon comparison (`Math.abs(a - b) < 0.001`).
- **Severity:** Low

### 4. `moveFileToFolder` clears undo/redo stacks silently
- **File:** `src/store/useCanvasStore.ts`
- **Issue:** Moving a file wipes all undo history without warning the user.
- **Fix:** Either preserve undo history or warn the user before clearing.
- **Severity:** Low

### 5. No file handle invalidation recovery
- **File:** `src/store/useCanvasStore.ts`
- **Issue:** If File System API permission is revoked (e.g., browser tab backgrounded too long), saves fail silently.
- **Fix:** Catch permission errors on save and prompt user to re-open folder.
- **Severity:** Low

### 6. `processedEdges` not memoized
- **File:** `src/App.tsx:625-630`
- **Issue:** `processedEdges` is recalculated on every render even when edges haven't changed.
- **Fix:** Wrap in `useMemo` with `canvas?.edges` as dependency.
- **Severity:** Low (performance)

### 7. TextNode image blob URLs leak on unmount
- **File:** `src/components/TextNode.tsx`
- **Issue:** `resolveImageUrl` creates blob URLs via `URL.createObjectURL()`, but when a TextNode is removed from the canvas, the blob URL is never revoked. Over time this leaks memory.
- **Fix:** Track resolved blob URLs and call `URL.revokeObjectURL()` in the MarkdownImage cleanup effect, or add a ref-counting mechanism to the image cache.
- **Severity:** Low (memory leak, only significant with many image nodes)

## UX Issues

### 8. No error display for users
- **Files:** `src/App.tsx`, `src/store/useCanvasStore.ts`
- **Issue:** The store sets `_error` state on failures (save errors, folder errors, etc.) but **no UI component ever renders it**. Users don't see save failures or other errors.
- **Fix:** Add a toast/notification component that subscribes to the store's `_error` state and auto-dismisses after a few seconds.
- **Severity:** High

### 9. No loading indicator
- **Files:** `src/App.tsx`, `src/store/useCanvasStore.ts`
- **Issue:** The store sets `_loading` state during async operations (openFolder, refreshFolder) but **no UI renders it**. Users get no feedback during potentially slow operations.
- **Fix:** Show a spinner or progress indicator when `_loading` is true.
- **Severity:** Medium

### 10. No keyboard shortcut help
- **Issue:** No in-app documentation listing available shortcuts. Users must discover them by accident.
- **Available shortcuts:** N/Shift+N (new node), Ctrl+C/V (copy/paste), Ctrl+Z/Shift+Z (undo/redo), Ctrl+S/Shift+S (save), Ctrl+A (select all), Ctrl+0 (fit view), Ctrl+B (toggle sidebar), Delete/Backspace (delete selected).
- **Fix:** Add a help modal (triggered by `?` or a toolbar button) showing all shortcuts.
- **Severity:** Medium

### 11. No React Error Boundary
- **File:** `src/App.tsx`
- **Issue:** If any component throws during render, the entire app crashes with a white screen and no recovery path.
- **Fix:** Wrap the app (or key subtrees) in an Error Boundary that shows a friendly error message with a "reload" button.
- **Severity:** Medium

### 12. Missing ARIA labels
- **Files:** `src/components/ClassNode.tsx`, `src/components/GroupNode.tsx`
- **Issue:** TextNode's textarea has `aria-label`, but ClassNode and GroupNode inputs do not. Context menu items and toolbar buttons (emoji icons) also lack accessible labels.
- **Fix:** Add `aria-label` attributes to all interactive elements.
- **Severity:** Low
